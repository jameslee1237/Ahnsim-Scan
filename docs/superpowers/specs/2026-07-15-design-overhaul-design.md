# 안심스캔 — 디자인 개편 (UI/UX Overhaul) 설계 문서

- Date: 2026-07-15
- Status: Draft (approved for planning)
- 선행 문서: v1/v2/result-caching/pwa-share-target 설계 문서들
- 승인된 시각 방향: 시각 컴패니언 목업 `A + C 믹스`(`.superpowers/brainstorm/.../home-merged.html`) — A(Calm Trust)의 차분한 블루+민트 색감 + C(Friendly Modern)의 모던·컴팩트 레이아웃, 여백을 줄여 밀도를 높인 버전.

## 1. 개요 (Overview)

현재 UI는 접근성·반응형·다크모드 등 기본기는 탄탄하지만, (1) 브랜드 정체성이 약하고(범용 실드 아이콘 + 평범한 헤딩, 히어로 아이콘 트리트먼트가 앱 아이콘과 불일치), (2) 홈 화면 상단에 회색 안내 박스(InstallGuide + PrivacyNotice)가 쌓여 주 작업(붙여넣기 → 분석)을 밀어내며, (3) 브랜드 컬러가 거의 드러나지 않아 "회색 + 약간의 파랑"으로 보이고, (4) 결과 카드가 가장 중요한 출력(판정·권장 조치)을 밋밋하게 보여준다.

이 개편은 로직·API·보안·PWA 동작을 전혀 건드리지 않고, **시각 언어와 정보 위계만** 재정비한다: 차분하면서 신뢰감 있는(안심) + 모던하고 컴팩트한 톤으로 통일하고, 안내 박스를 정리하며, 결과 카드의 위계를 강화하고, 아이콘/히어로 트리트먼트를 앱 아이콘과 일치시킨다.

## 2. 목표 / 비목표

**목표**:
- 일관된 시각 정체성: 브랜드 블루 `#2563eb`(살짝 밝은 톤) + 민트/소프트 서피스 + 따뜻한 오프화이트 배경. **라이트 모드 전용**.
- 브랜드 색 완전 통일: `--primary`·히어로·버튼·PWA 매니페스트(`theme_color`)·`viewport.themeColor`·PNG 아이콘을 모두 `#2563eb`로 일치(**PNG 아이콘 재생성 포함**).
- 정보 위계 강화, 특히 결과 카드(큰 판정 헤더 + 강조된 권장 조치 콜아웃).
- 홈 상단 회색 박스 정리: 두 개의 안내 박스를 한 줄짜리 슬림 안내 + 접이식 상세로 축약(기존 정보는 모두 보존).
- 아이콘/히어로 트리트먼트를 앱 아이콘(라운드 사각형, 그라디언트 실드)과 일치.
- 다크 모드 제거(라이트 전용). 접근성·반응형은 현행 수준 이상 유지(신규 색상 WCAG AA 대비 충족).
- **새 npm 의존성 없이**, 기존 shadcn 컴포넌트 + Tailwind v4 토큰(`globals.css`) 재스타일로 구현.

**비목표**:
- 분석 파이프라인/프롬프트/`/api/analyze`/보안(Turnstile·rate limit·quota·cache)/PWA share-target 동작 변경 — 전혀 손대지 않는다.
- 새 폰트/아이콘 라이브러리 도입 및 새 npm 의존성 — 없음. (기존 PWA PNG 아이콘은 새 브랜드 색 `#2563eb`에 맞춰 재생성하되, 새 도구/라이브러리 없이 기존 `qlmanage`+`sips` 방식으로.)
- 카피(문구)에 근거 없는 성능 주장 추가(예: "3초" 같은 하드 클레임) — 목업의 해당 문구는 자리표시자이며, 정확한 카피로 대체(§9의 카피 확인 항목).
- 컴포넌트의 동작/props/공개 API 변경 — 시각 스타일과 마크업 구조만 조정하고 기존 상태·핸들러·prop 계약은 유지.

## 3. 디자인 방향 (Visual Language)

**색상(라이트 전용)**: 브랜드 프라이머리 `#2563eb`(그라디언트 `#2563eb → #1e5fd0`)를 히어로 아이콘 타일과 주요 버튼에 사용. 서피스는 따뜻한 오프화이트(예: 페이지 배경 `#f6f8fc`), 카드 흰색, 소프트 블루 틴트(`#eef4ff`)를 슬림 안내에. 판정 색(안전 green / 의심 amber / 위험 red)은 의미 유지하되 헤더 트리트먼트 강화. 다크 모드는 제거한다(§6).

**브랜드 블루 통일(결정됨)**: 브랜드 블루는 `#2563eb` 하나로 통일한다. 구현 시 (1) `globals.css`의 `--primary`(및 `--ring`)를 `#2563eb`로, (2) PWA 매니페스트 `theme_color`와 `layout.tsx`의 `viewport.themeColor`를 `#2563eb`로, (3) 4개 PNG 아이콘(192/512, any+maskable)을 `#2563eb` 배경으로 **재생성**한다(기존 `qlmanage`+`sips` 방식, 새 의존성 없이). 이로써 앱 아이콘·매니페스트·인앱 히어로·버튼 색이 완전히 일치한다.

**형태**: 라운드는 컴팩트-모던 감각으로 카드 14–16px, 버튼 12–13px, 작은 요소 8–10px 수준(현행 `--radius: 1rem`에서 과하지 않게 정리). 그림자는 소프트·낮은 확산.

**여백/리듬**: A의 과한 여백을 줄여 세로 밀도를 높인다(섹션 간 간격을 촘촘히). 모바일 우선.

**타이포**: 폰트는 현행 Noto Sans KR 유지(새 폰트 없음). 위계는 굵기/크기로: 워드마크·판정은 굵고 크게(800–900), 본문은 명료한 크기, 보조 텍스트는 muted. 결과 카드에서 전부 `text-sm muted`로 평평하던 것을 개선.

## 4. 컴포넌트별 변경

모두 시각/마크업만 조정하며 동작·props는 유지한다.

**4.1 디자인 토큰 (`src/app/globals.css`)**
- `--primary`를 `#2563eb`로 조정(그리고 `--ring` 동기화). 프라이머리 그라디언트용 보조 변수(예: `--primary-strong: #1e5fd0`) 추가 가능.
- 따뜻한 배경/서피스/소프트-블루 틴트/민트 액센트 토큰 추가(라이트 전용). 순수 무채색 위주였던 팔레트에 약간의 채도(웜/블루 틴트) 부여.
- 다크 모드 제거: `.dark { … }` 토큰 블록을 삭제한다. 단, `@custom-variant dark (&:is(.dark *))`(클래스 기반)는 **유지**한다 — 삭제하면 `dark:`가 OS `prefers-color-scheme` 미디어로 되돌아가 생성된 shadcn `ui/*`의 남은 `dark:` 유틸리티가 OS 다크에서 활성화된다. 유지하면 `.dark` 클래스가 없어 전부 무력화된다. 재스타일하는 앱 컴포넌트(ResultCard/AnalyzeForm)의 `dark:`는 제거, `ui/*`의 `dark:`는 무해하게 남긴다(§6).
- 판정 색은 컴포넌트 로컬 클래스(ResultCard)에서 계속 관리.

**4.2 히어로 (`src/app/page.tsx`)**
- 파란 원 + 얇은 실드 → **라운드 사각형 그라디언트 타일 + 흰색 실드-체크**(앱 아이콘과 동일 언어) + "안심스캔" 워드마크를 한 줄 락업으로. 아래 한 줄 태그라인("문자·이메일·스크린샷, 사기인지 바로 확인"). 상하 여백 축소.
- 상단 그라디언트 워시는 유지하되 살짝 더 존재감 있게(과하지 않게).

**4.3 안내 정리 (`PrivacyNotice` + `InstallGuide` 통합/축약)**
- 두 개의 회색 박스 → **한 줄 슬림 안내 바**: 좌측 "🔒 입력 내용은 저장하지 않아요"(개인정보 요지), 우측 "홈 화면에 추가 ›"(설치, `beforeinstallprompt` 있을 때만/데스크톱 안내). 
- 상세(설치 단계, 사용 방법, iOS 미지원, 무료 사용량 한도)는 **접이식 "자세히"** 또는 작은 상세 영역으로 이동 — 기존 정보는 하나도 잃지 않고 위계만 낮춘다.
- `InstallButton`의 `beforeinstallprompt` 로직은 그대로 유지, 시각만 슬림 안내에 맞춤. iOS 미지원 문구·사용량 한도 안내도 상세 영역에 유지.

**4.4 입력 폼 (`AnalyzeForm` + `ImageUploader`)**
- 탭을 모던 세그먼티드 컨트롤(라이트 트랙 + 활성 흰색/프라이머리 텍스트)로 재스타일. 기존 `Tabs`(shadcn) 마크업/동작 유지, 클래스만.
- 입력/텍스트에어리어/카운터/Turnstile 컨테이너/제출 버튼을 새 팔레트·라운드·간격으로 재스타일. 제출 버튼은 프라이머리 그라디언트. 모바일 sticky 제출 버튼 동작 유지.
- `ImageUploader` 드롭존을 동일 언어로 재스타일(동작·검증 로직 불변).

**4.5 결과 카드 (`ResultCard`)** — 위계 핵심 개선
- 상단 **판정 헤더**: 판정색 틴트 배경 + 라운드 아이콘 타일(해당 판정 아이콘) + 큰 판정 텍스트("위험/의심/안전", 800–900) + 한 줄 평이한 부제(예: "사기일 가능성이 매우 높아요") + 우측에 큰 위험 점수(N/100).
- 그 아래: progress(판정색), "판독된/분석한 메시지"(evidence 하이라이트 유지 — `highlightEvidence` 그대로), "주요 위험 신호" 목록.
- **권장 조치("이렇게 하세요")를 강조 콜아웃**(틴트 박스)으로 승격 — 현재 맨 아래 조용한 줄에서 핵심 위계로. `explanation`은 그 위/근처에 배치.
- "결과 지우기"는 조용한 보조 버튼으로. 헤딩(h2/h3) 구조·`role/aria-live`·`mark` 하이라이트 등 접근성 마크업 유지.

**4.6 공통 요소**
- 버튼/인풋/뱃지/구분선 등 shadcn 프리미티브는 유지하되 새 토큰으로 자연히 통일. 개별 오버라이드는 최소화하고 토큰 레벨에서 최대한 해결.

**4.7 아이콘 · 매니페스트**
- 4개 PNG 아이콘(192/512, any+maskable)을 `#2563eb` 배경으로 재생성(기존 `qlmanage`+`sips` 방식, 새 의존성 없이). 매니페스트 `theme_color`와 `layout.tsx`의 `viewport.themeColor`를 `#2563eb`로 갱신. 이 색은 §3의 브랜드 블루와 반드시 일치.

## 5. 접근성 (Accessibility)

현행 수준을 유지·강화한다: `aria-live` 결과 알림, `prefers-reduced-motion` 처리, 올바른 헤딩 계층(h1→h2→h3), 포커스 링, `aria-hidden` 아이콘, 좁은 화면 줄바꿈(min-w-0). **신규 색 조합은 텍스트/배경 대비 WCAG AA(본문 4.5:1, 큰 텍스트 3:1) 충족을 확인**한다(특히 판정 틴트 위 텍스트, muted 텍스트, 프라이머리 버튼 텍스트). 키보드 내비게이션·탭 순서 유지.

## 6. 다크 모드 제거

다크 모드를 제거하고 **라이트 모드 전용**으로 만든다. 현재 다크 모드는 클래스 기반(`.dark`)이며 이를 켜는 코드가 없어 사실상 비활성 상태다. 정리 범위: `globals.css`의 `.dark { … }` 토큰 블록 삭제; `@custom-variant dark (&:is(.dark *))`는 **유지**(아래 주의); 재스타일하는 앱 컴포넌트의 `dark:` 클래스 제거(예: `ResultCard`의 판정 뱃지·`mark` 하이라이트, `AnalyzeForm`의 amber 카운터). **주의**: `@custom-variant dark`를 삭제하면 Tailwind v4에서 `dark:`가 `prefers-color-scheme` 미디어 기반으로 되돌아가, 생성된 shadcn `ui/*`에 남은 `dark:` 유틸리티가 OS 다크 설정에서 활성화되어 깨진 부분 다크 모드가 된다. 클래스 기반 변수를 유지하고 `.dark` 클래스를 어디서도 추가하지 않으면 그 유틸리티들은 전부 무력화된다. 삭제 후 라이트 팔레트만 남고 렌더링이 동일하게 유지되는지 확인한다.

## 7. 반응형

모바일 우선. 컴팩트 간격이 작은 화면에서 자연스럽게 스케일. 결과 카드 판정 헤더는 좁은 폭에서 줄바꿈/축소 대응. 모바일 sticky 제출 버튼 유지.

## 8. 범위 밖(불변) 로직

`src/lib/**`(분석·보안·캐시·downscale), `/api/analyze`, `/share-target`, 프롬프트, Zod 스키마, 프로바이더, PWA(manifest/sw/공유 수신 로직)의 **동작은 변경하지 않는다**. `page.tsx`의 공유 수신(`?shared=1`) 로직과 `AnalyzeForm`의 상태/제출/Turnstile 로직은 그대로 두고 마크업/클래스만 조정한다.

## 9. 테스트 / 검증

- 기존 자동화 테스트(`src/lib/**`, `/api/analyze`, `/share-target`)는 로직 불변이므로 그대로 통과해야 한다 — 회귀 없음 확인(`pnpm test`).
- 클라이언트/시각 컴포넌트는 이 프로젝트 관례대로 Vitest 없이 수동 검증.
- 매 단계 `npx tsc --noEmit`, `pnpm lint`, 최종 `pnpm build` 통과.
- **수동 시각 QA 체크리스트**: 모바일/데스크톱(라이트 전용); 판정 3종(안전/의심/위험) 결과 카드; 이미지 모드 결과(extractedText 표시); 공유 채우기(`?shared=1`) 후 폼 정상; Turnstile·분석 정상; 슬림 안내 상세 펼침·설치 버튼(가능 환경)·iOS 문구·사용량 한도 문구 존재; 매니페스트 `theme_color`·PNG 아이콘 색이 `#2563eb` 반영.
- **카피(확정)**: 히어로 태그라인은 **"문자·이메일·스크린샷, 사기인지 바로 확인"**으로 확정(목업의 "3초 만에…"는 폐기). 결과 카드의 판정 부제(예: "사기일 가능성이 매우 높아요")는 판정별 평이한 한 줄로 구현 시 확정.

## 10. 구현 접근

`frontend-design` 스킬을 구현 단계에서 활용한다. 토큰(`globals.css`)부터 정리한 뒤 컴포넌트를 위→아래(히어로 → 안내 → 폼 → 결과 카드 → 업로더) 순으로 재스타일하며, 각 단계에서 tsc/lint/시각 확인. 새 의존성 없이 기존 컴포넌트 마크업/클래스 수정으로 진행한다.

## 11. 이후(범위 밖)

- PWA 앱 아이콘(PNG) 리프레시(원한다면 별도 소규모 작업).
- 마이크로 인터랙션/모션 정교화(핵심 개편 이후 선택).
